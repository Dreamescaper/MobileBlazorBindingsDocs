Backend-specific services

Important

This project is an experimental release. We hope you try out Experimental Mobile Blazor Bindings and provide feedback at https://github.com/xamarin/MobileBlazorBindings.

While many service implementations used in dependency injection have a common implementation for all platforms, there are cases where a platform-specific service implementation needs to be supplied.

Registering backend-specific services involves a few steps:

  1. Define a common interface in the shared project. For example, the Todo App sample defines an ITextToSpeech interface.

  2. Implement the interface in each applicable backend-specific project. It's OK for some backends to not have an implementation. The Todo App sample has an Android implementation and an iOS implementation.

  3. Register the backend-specific implementation in each backend project. In an Android backend, this is done in the MainActivity class OnCreate method, and in an iOS backend, this is done in the AppDelegate class FinishedLaunching method.

  4. In the shared project change the App class to take in a constructor parameter IServiceCollection additionalServices, as seen here.

  5. Change the call to ConfigureServices to register the backend-specific services with the AddAdditionalServices method, as seen here.

  6. Now you can consume the service as you would consume any other registered service from the dependency injection container. Some examples are shown in the dependency injection topic. If the service is an optional service because it's not available on all platforms, have the consuming code take in a list of the service, and it will receive either zero instances of the service (if not available) or one instance (if available).
